1. Data Structure — Vocabulary Array in GAME_DATA
Recommend a flat array of word records under KC.data.archive.vocab. Each record carries its own audio key bundle so mission code stays asset-agnostic (no string-building inside the loop — keys are looked up, never synthesized, per the Audio Asset Key Contract).
```
// kc_data.js  (additive only; do not refactor existing GAME_DATA)
KC.data.archive = {
  vocab: [
    {
      id: 'cat',
      word: 'CAT',                  // canonical UPPERCASE; per-letter targets
      tier: 1,                      // 1=3-letter, 2=4, 3=5 (gated progression)
      audio: {
        cue:    'arc_cat_cue',      // diegetic SFX (the meow)
        prompt: 'amelia_spell_cat', // VO: "Spell Cat."
        intro:  'amelia_arc_cat_in',// optional flavor lead-in
        success:'arc_cat_win',      // word-complete fanfare
      }
    },
    // ...
  ],
  // Per-letter feedback is GLOBAL, not per-word — keeps the bank lean.
  letterFeedback: {
    correct: 'arc_letter_ok',       // short triumph chime (Web Audio synth ok)
    wrong:   'arc_letter_buzz',     // gentle buzz (Web Audio synth ok)
  },
  // Spatial nudge VO bank — see §2 for how these are selected.
  proximityVO: {
    // distance buckets
    adjacent: { L:'amelia_one_left',  R:'amelia_one_right',
                U:'amelia_one_up',    D:'amelia_one_down' },
    near:     { L:'amelia_two_left',  R:'amelia_two_right',
                U:'amelia_two_up',    D:'amelia_two_down' },
    far:      'amelia_far_off',     // > 3 units: "Way off. Listen again."
    sameRow:  'amelia_same_row',    // tie-breaker hint
  }
};
```
Why this shape:
VO files (`prompt`, `intro`, success-fanfare) route through HTML5 `
audio_engine.md (/c:/nbs/kc/.rules/audio_engine.md)
).
`letterFeedback.correct`/`wrong` are tiny, latency-critical → Web Audio synth or pre-decoded buffers.
Keys are stored verbatim — never normalized, never lowercased on the fly.
2. The Proximity Matrix — 2D Coordinate Dictionary
A static lookup, defined once. Y increases downward (matches physical row layout). X is fractional to model the half-key row stagger (this is what makes "F is one left of G" feel right while "X→C" reads as diagonal).
```
// kc_data.js
KC.data.archive.keyGrid = {
  // Row 1 (numbers) y=0
  '1':[0.0,0], '2':[1.0,0], '3':[2.0,0], '4':[3.0,0], '5':[4.0,0],
  '6':[5.0,0], '7':[6.0,0], '8':[7.0,0], '9':[8.0,0], '0':[9.0,0],
  // Row 2 (QWERTY) y=1, +0.5 stagger
  Q:[0.5,1], W:[1.5,1], E:[2.5,1], R:[3.5,1], T:[4.5,1],
  Y:[5.5,1], U:[6.5,1], I:[7.5,1], O:[8.5,1], P:[9.5,1],
  // Row 3 (Home) y=2, +0.75
  A:[0.75,2], S:[1.75,2], D:[2.75,2], F:[3.75,2], G:[4.75,2],
  H:[5.75,2], J:[6.75,2], K:[7.75,2], L:[8.75,2],
  // Row 4 (Bottom) y=3, +1.25
  Z:[1.25,3], X:[2.25,3], C:[3.25,3], V:[4.25,3],
  B:[5.25,3], N:[6.25,3], M:[7.25,3],
};
// Pure helper — lives on the mission, not the global namespace.
KC.mission.archive._distance = function(pressed, target) {
  const a = KC.data.archive.keyGrid[pressed];
  const b = KC.data.archive.keyGrid[target];
  if (!a || !b) return { bucket:'far', dx:0, dy:0 }; // S1: strict, no coercion
  const dx = b[0] - a[0];   // +x means target is to the RIGHT of pressed
  const dy = b[1] - a[1];   // +y means target is DOWN from pressed
  const cheb = Math.max(Math.abs(dx), Math.abs(dy));
  let bucket = 'far';
  if (cheb <= 1.0)      bucket = 'adjacent';
  else if (cheb <= 2.0) bucket = 'near';
  return { bucket, dx, dy };
};
```
Direction selection rule (deterministic, whitelist-friendly per S1):
If `|dy| > |dx|` → vertical: `dy>0 ? 'D' : 'U'`
Else → horizontal: `dx>0 ? 'R' : 'L'`
`bucket==='far'` short-circuits to `proximityVO.far` regardless of vector.
System-modifier keys are never in keyGrid, which structurally enforces the curriculum ban (no Shift/Ctrl/Alt entries to map against).
3. State Machine — this.state Critical States
A single string-valued this.state (S1-friendly: switch on ===). Slow-paced, so we need explicit listening gates — no implicit "always-on" input.
State Meaning Input accepted?
IDLE Pre-start / between words No
PROMPTING Playing cue → prompt → "go" tone No (S4 still gates e.repeat)
LISTENING Awaiting next letter of current word Yes (target letter only matters for grading)
FEEDBACK_OK Playing correct chime, advancing pointer No
FEEDBACK_WRONG Playing buzz + spatial VO No
WORD_COMPLETE Success fanfare, pre-roll next word No
EXITING S6 killswitch in flight No
Companion fields on the mission instance:
```
this.state         = 'IDLE';
this.dictationId   = null;          // S2 token
this.timeouts      = [];            // cleared per async_state.md
this.wordIndex     = 0;             // pointer into vocab[]
this.letterIndex   = 0;             // pointer into vocab[wordIndex].word
this.currentWord   = null;          // cached vocab record
```
Flow:
```
start() → state=PROMPTING → playCue → playPrompt → state=LISTENING
   ↑                                                       │
   │                                                       ▼
WORD_COMPLETE ← (letterIndex===word.length) ← FEEDBACK_OK ←┤ (correct key)
                                                           │
                              FEEDBACK_WRONG ──────────────┘ (wrong key)
                                     │
                                     ▼
                              state=LISTENING (re-arm; same letterIndex)
```
The wrong-key branch does not advance letterIndex — pacing is zero-pressure, the user retries until correct. Re-arming back to LISTENING happens inside the chained-VO completion callback, gated by the dictation token.
4. Sentinel Compliance — S2 (Async Tokens) & S4 (Ghost-Key Filter)
S4 — Ghost-Key Filter (the easy one, but easy to get wrong)
```
KC.handlers.archive = {
  handleInput(e) {
    if (e.repeat) return;                          // S4: ABSOLUTE first gate
    if (KC.mission.archive.state !== 'LISTENING')  // S1: strict equality
        return;
    KC.mission.archive._grade(e.key);
  }
};
```
Critical points:
Diagnostic keys (PageUp etc.) live **after** the `e.repeat` gate — no exceptions.
Esc-to-exit is also gated by the state machine; it triggers the S6 killswitch, not a direct hub jump.
The state guard means held arrow keys, screen-reader review-cursor chatter, and modifier latches all silently drop while VO is playing.
S2 — Async Token (dictationId) for the chained VO + retry loop
The danger here is real: a slow user may press a wrong key while the prompt VO is still finishing, or press a correct key during the wrong-key spatial-feedback VO. Without tokens, two callback chains race and Amelia talks over herself.
```
_speakAndArm(currentDictation) {
  if (this.dictationId !== currentDictation) return;     // S2 gate #1
  KC.audio.playVO(this.currentWord.audio.cue, () => {
    if (this.dictationId !== currentDictation) return;   // S2 gate #2
    KC.audio.playVO(this.currentWord.audio.prompt, () => {
      if (this.dictationId !== currentDictation) return; // S2 gate #3
      this.state = 'LISTENING';
    });
  });
},
_grade(key) {
  const target = this.currentWord.word[this.letterIndex];
  if (key.toUpperCase() === target) {                    // S1: explicit ===
    this._onCorrect();
  } else {
    this._onWrong(key.toUpperCase(), target);
  }
},
_onWrong(pressed, target) {
  this.state = 'FEEDBACK_WRONG';
  const tok = ++this.dictationId === null ? Date.now() : Date.now();
  this.dictationId = tok;                                // invalidates prior chain
  KC.audio.playSFX(KC.data.archive.letterFeedback.wrong);
  const { bucket, dx, dy } = KC.mission.archive._distance(pressed, target);
  const voKey = this._pickProximityVO(bucket, dx, dy);
  const t = setTimeout(() => {
    if (this.dictationId !== tok) return;                // S2 gate
    KC.audio.playVO(voKey, () => {
      if (this.dictationId !== tok) return;              // S2 gate
      this.state = 'LISTENING';                          // re-arm
    });
  }, 250);
  this.timeouts.push(t);
},
```
Why a fresh token on every wrong-press: if the user mashes two wrong keys quickly, the second press invalidates the first VO chain mid-flight, so we never get "Move one key left" overlapping with "Move two keys right." The same token-bump is used on _onCorrect and on word advance.
